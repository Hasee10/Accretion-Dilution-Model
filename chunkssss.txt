# CHUNK 1 — Database & Auth Foundation (Supabase)

**Paste this prompt to Antigravity:**

```
I am building a professional open-source quantitative finance platform called "QuantEdge" 
deployed on Vercel. Replace the current SQLite + Clerk setup with a production-grade 
Supabase backend. Do ALL of the following:

## 1. Supabase Schema (run in Supabase SQL editor)

Create the following tables with Row Level Security enabled on all of them:

### users (extends Supabase auth.users)
CREATE TABLE public.profiles (
  id UUID REFERENCES auth.users(id) ON DELETE CASCADE PRIMARY KEY,
  full_name TEXT NOT NULL,
  email TEXT NOT NULL,
  company TEXT,
  role TEXT DEFAULT 'analyst',
  avatar_url TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

### saved_deals (Merger Analysis saves)
CREATE TABLE public.saved_deals (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID REFERENCES public.profiles(id) ON DELETE CASCADE NOT NULL,
  deal_name TEXT NOT NULL,
  deal_data JSONB NOT NULL,
  result_snapshot JSONB,
  is_public BOOLEAN DEFAULT FALSE,
  tags TEXT[],
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

### dcf_models (DCF saves)
CREATE TABLE public.dcf_models (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID REFERENCES public.profiles(id) ON DELETE CASCADE NOT NULL,
  model_name TEXT NOT NULL,
  ticker TEXT,
  model_data JSONB NOT NULL,
  result_snapshot JSONB,
  is_public BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

### watchlist
CREATE TABLE public.watchlist (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID REFERENCES public.profiles(id) ON DELETE CASCADE NOT NULL,
  ticker TEXT NOT NULL,
  company_name TEXT,
  added_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, ticker)
);

### ai_chat_history
CREATE TABLE public.ai_chat_history (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID REFERENCES public.profiles(id) ON DELETE CASCADE NOT NULL,
  session_id UUID NOT NULL,
  role TEXT CHECK(role IN ('user', 'assistant')) NOT NULL,
  content TEXT NOT NULL,
  context_type TEXT, -- 'merger', 'dcf', 'general'
  context_data JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

Enable RLS on every table. Create policies so users can only read/write their own rows.
For is_public=true rows in saved_deals and dcf_models, allow all authenticated users to SELECT.

## 2. Supabase Auth Configuration

In the Supabase dashboard:
- Enable Email provider
- Enable "Confirm email" 
- Set site URL to NEXT_PUBLIC_APP_URL
- Set redirect URLs to include /auth/callback

## 3. Frontend Auth Pages (replace Clerk completely)

Create these pages using Supabase Auth JS client (@supabase/supabase-js, @supabase/ssr):

### /auth/sign-in
- Email + Password form
- "Remember me" checkbox
- Forgot password link → /auth/forgot-password
- Sign up link → /auth/sign-up
- Show loading spinner on submit
- On success, redirect to /dashboard
- Show inline error messages (invalid credentials, email not confirmed)

### /auth/sign-up  
- Full Name, Email, Password, Confirm Password, Company (optional)
- Password strength indicator (weak/medium/strong based on length + special chars)
- On submit: create Supabase auth user, then insert into public.profiles
- Show success screen: "Check your inbox — we sent a confirmation link to {email}"
- Do NOT redirect yet; wait for email confirmation

### /auth/forgot-password
- Email field only
- Call supabase.auth.resetPasswordForEmail() with redirectTo: /auth/reset-password
- Show success message regardless of whether email exists (security)

### /auth/reset-password  
- New Password + Confirm Password
- Call supabase.auth.updateUser({ password })
- Redirect to /auth/sign-in on success

### /auth/callback
- Handle the OAuth/magic link callback
- supabase.auth.exchangeCodeForSession(code)
- Redirect to /dashboard

## 4. Email Templates (in Supabase Dashboard → Auth → Email Templates)

Customize all templates to match QuantEdge branding:
- Confirmation email: Subject "Confirm your QuantEdge account"
- Password reset: Subject "Reset your QuantEdge password"  
- Magic link: Subject "Your QuantEdge sign-in link"
Each should have a clean dark-themed HTML email with the QuantEdge name prominent.

## 5. Session Management

Create a Supabase client singleton at lib/supabase/client.ts (browser) and 
lib/supabase/server.ts (server-side using @supabase/ssr createServerClient with cookies).

Create middleware.ts at root using @supabase/ssr to:
- Refresh sessions on every request
- Protect all routes under /dashboard, /merger-analysis, /dcf-valuation
- Redirect unauthenticated users to /auth/sign-in with ?redirect= param
- After sign-in, redirect back to original URL

## 6. Profile Page (/settings/profile)

Form fields: Full Name, Company, Role (dropdown: Analyst, Associate, VP, Director, MD)
Avatar upload using Supabase Storage bucket "avatars" (public bucket, 2MB limit, image only)
Show current email (read only, with "Change email" link — not yet implemented, just show it)
Save button calls supabase.from('profiles').update(...)
Show toast on success/error

Install packages:
- @supabase/supabase-js
- @supabase/ssr
- Remove all @clerk references

## Backend (Python FastAPI) — app/core/lbo_logic.py

Create an LBOModel class with this calculation logic:

### Inputs
- entry_ev: float (Entry Enterprise Value, $M)
- entry_ebitda: float (Entry EBITDA, $M) → derives Entry Multiple
- exit_multiple: float (EV/EBITDA at exit)
- hold_period: int (years, 3-7)
- debt_pct: float (% of EV funded by debt, typically 0.5-0.7)
- interest_rate: float (blended rate on debt)
- ebitda_growth_rate: float (annual EBITDA growth)
- revenue_at_entry: float (for reference)
- management_fee_pct: float (PE firm annual fee as % of equity, default 0.02)

### Calculations
```python
def calculate(self):
    # Entry
    entry_equity = entry_ev * (1 - debt_pct)
    entry_debt = entry_ev * debt_pct
    entry_multiple = entry_ev / entry_ebitda
    
    # Exit (after hold_period years)
    exit_ebitda = entry_ebitda * ((1 + ebitda_growth_rate) ** hold_period)
    exit_ev = exit_ebitda * exit_multiple
    
    # Debt paydown (simplified: assume 10% of EBITDA per year goes to debt paydown)
    annual_debt_paydown = entry_ebitda * 0.10  # simplified
    remaining_debt = max(0, entry_debt - (annual_debt_paydown * hold_period))
    
    # Interest paid (impacts debt paydown indirectly — simplify as interest on avg debt)
    avg_debt = (entry_debt + remaining_debt) / 2
    total_interest = avg_debt * interest_rate * hold_period
    
    # Exit equity
    exit_equity = exit_ev - remaining_debt
    
    # Returns
    moic = exit_equity / entry_equity  # Multiple of Invested Capital
    irr = ((exit_equity / entry_equity) ** (1 / hold_period)) - 1
    
    # Management fees drag
    annual_fee = entry_equity * management_fee_pct
    fee_drag_on_irr = (annual_fee * hold_period) / entry_equity / hold_period  # rough drag
    
    # Sensitivity: IRR across exit multiples (exit_multiple ± 2) and hold periods (3,4,5,6,7)
    sensitivity = {}
    for hp in [3, 4, 5, 6, 7]:
        sensitivity[hp] = {}
        for em in [exit_multiple - 2, exit_multiple - 1, exit_multiple, exit_multiple + 1, exit_multiple + 2]:
            s_exit_ev = exit_ebitda * em  # use same exit_ebitda
            s_exit_equity = s_exit_ev - remaining_debt
            s_moic = s_exit_equity / entry_equity
            s_irr = (s_moic ** (1/hp)) - 1
            sensitivity[hp][em] = round(s_irr * 100, 1)
    
    return {
        "entry_multiple": round(entry_multiple, 1),
        "entry_equity": round(entry_equity, 1),
        "entry_debt": round(entry_debt, 1),
        "exit_ebitda": round(exit_ebitda, 1),
        "exit_ev": round(exit_ev, 1),
        "exit_equity": round(exit_equity, 1),
        "remaining_debt": round(remaining_debt, 1),
        "moic": round(moic, 2),
        "irr": round(irr * 100, 1),
        "sensitivity": sensitivity  # {hold_period: {exit_multiple: irr%}}
    }
```
